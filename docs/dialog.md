现在的程序包括了前端的控制面板和后端的按键映射服务，这样不好，因为前端的GUI其实是比较消耗系统资源的，我要求前后端分离，也就是说，把当前的程序分割为一个包含UI的控制面板程序和一个后台Worker/Service程序，这样：打开界面时，连接后台，关掉界面时，UI进程直接退出，后台服务单独保留。而且service程序的启动与否也由前端命令面板控制，命令面板可以控制service开机自启动以及当前退出与否（设计两个radio button，之前说是放在标题栏，现在衡量之后还是放在控制面板吧）。
